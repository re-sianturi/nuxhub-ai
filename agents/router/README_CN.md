# 路由代理 — 使用指南与商业成果

> **这不是技术手册。** 这是一份面向商业用户的友好指南，解释："路由代理能为我的业务做什么，以及如何使用它？"

---

## 1. 路由代理是什么？（简单语言）

想象你在线上商店的前台有一位**智能接待员**。每次有人发消息时，这位接待员不会立即推销。相反，它会先倾听，然后决定：

- **"这个人真的想买"** → 转给销售
- **"这个人生气了"** → 转给经理
- **"这是垃圾信息发送者"** → 直接踢出，不回复
- **"这个人之前聊过但消失了"** → 转给跟进团队

**路由代理 = 那位智能接待员。** 它坐在前门，检查每个进来的人，然后将他们引导到正确的路径。

---

## 2. 商业成果 — 它能给你的业务带来什么？

### 2.1. 销售团队不再浪费时间处理垃圾信息

| 使用路由代理之前 | 使用路由代理之后 |
|---|---|
| 销售团队回复垃圾信息发送者："需要快速贷款吗？" | 垃圾信息发送者自动被屏蔽，销售团队看不到 |
| 团队精力浪费在恶作剧聊天上 | 恶作剧聊天被引导回商业话题，不干扰销售 |
| 20% 的销售时间浪费在无用的聊天上 | 销售团队 100% 专注于真正想购买的潜在客户 |

**成果：** 销售团队生产力提升 20-30%。他们与真正的买家聊天，而不是垃圾信息发送者。

### 2.2. 生气的潜在客户不再收到机器人回复

| 之前 | 之后 |
|---|---|
| 投诉"退款！"收到回复："我们有10%折扣促销" | 投诉立即转给人工经理 |
| 生气的人因为机器人回复而更生气 | 生气的人由人工以同理心处理 |
| 在 Google / Instagram 上留下差评 | 评价保持正面，因为问题被认真对待 |

**成果：** 品牌声誉得到保护。生气的潜在客户由人工处理，而不是被不懂情感的 AI 推送。

### 2.3. 之前聊过但没购买的潜在客户 — 不会流失

| 之前 | 之后 |
|---|---|
| 某人昨天聊天，问了价格，然后消失了。没有跟进。 | 路由代理记得他们。如果他们再次聊天，从上次中断的地方继续，不会从头再问"您叫什么名字？"。 |
| 60% 的潜在客户只是问问，没有培育系统 | 冷门潜在客户被保存，自动在 1小时/3小时/7小时/30天 后跟进 |
| 潜在客户数据库闲置不用 | 每个潜在客户都有 CRM 文件，完整历史记录，随时可以接续 |

**成果：** 转化率提高。"只是问问"的潜在客户被培育，而不是被丢弃。销售不需要每次聊天都从零开始。

### 2.4. 不再"错误转接"给销售

| 之前 | 之后 |
|---|---|
| 某人要求企业定制方案，初级销售回复了（他没有权限） | 企业定制方案直接转给销售经理，不是初级销售 |
| 退款请求由无法退款的营销团队处理 | 退款请求直接转给 billing 团队 |
| 法律威胁由愚蠢的聊天机器人回复 | 法律威胁直接转给法务团队，不会有错误回复 |

**成果：** 减少财务和法律损失。大交易由有权限的人处理。敏感问题不会被错误处理。

### 2.5. 每个潜在客户都获得与其"质量"匹配的处理

路由代理给他们贴上标签：
- **热门潜在客户** → 直接转给销售，不要等待
- **温暖潜在客户** → 培育、跟进、提供内容
- **冷门潜在客户** → 保存、稍后跟进、不要推动
- **垃圾信息** → 屏蔽、不回复
- **生气** → 人工、立即处理

**成果：** 自动销售优先级排序。不会因为销售忙于回复垃圾信息而让热门潜在客户被遗漏。

## 3. 路由代理如何工作（类比）

### 类比：医院

有人进入医院 → 路由代理 = **急诊室门口的分诊护士**。

1. **首先**，检查："这个人是紧急情况吗？大出血？" → 急诊室
2. **第二**，检查："这个人携带病毒吗？" → 隔离区
3. **第三**，检查："这个人只是问药房方向吗？" → 指引他们
4. **第四**，检查："这个人是回来复查的吗？" → 继续他们的病历记录

路由代理工作方式相同。每条进来的聊天：
1. **垃圾信息检查** → 踢出还是继续？
2. **升级检查** → 生气/法律/退款？
3. **回访检查** → 之前聊过？
4. **路由** → 引导到正确的代理

## 4. 路由代理处理的 5 种场景

### 场景 A：正常的新潜在客户

**输入：**
```
用户："你好，我需要帮助获取更多客户。你能帮忙吗？"
```

**路由代理工作：**
1. 垃圾信息评分？0 → 干净
2. 升级？没有生气关键词 → 继续
3. 回访潜在客户？从未聊过 → 新客户
4. 决策：**发现（DISCOVERY）** → 委托给发现代理

**输出给用户：**
```
代理："当然可以，先生。请问您目前经营的是什么业务？"
```

**商业价值：** 潜在客户进入销售管道，开始资格评估。

---

### 场景 B：垃圾信息发送者

**输入：**
```
用户："需要快速贷款？低利率0.1%立即转账到账户1234567890!!!"
```

**路由代理工作：**
1. 垃圾信息评分？110（跑题 +10，诈骗模式 +50，紧急操纵 +50）
2. 垃圾信息评分 ≥ 100 → **黑名单（BLACKLIST）**
3. 不发送任何回复
4. 保存到黑名单

**输出给用户：**
```
（无回复）
```

**商业价值：** 销售不受干扰。没有团队浪费时间回复垃圾信息发送者。

---

### 场景 C：生气的人 / 投诉

**输入：**
```
用户："我很生气！服务太差了，我要退款！"
```

**路由代理工作：**
1. 垃圾信息评分？0 → 不是垃圾信息
2. 升级？关键词"生气" + "退款" → **升级 = 真**
3. 优先级：高
4. 决策：**人工升级（HUMAN_ESCALATION）** → 委托给人工，不是 AI

**输出给用户：**
```
代理："对于给您带来的不便，我们深表歉意。我将为您转接能够直接协助的团队。"
```

**商业价值：** 敏感问题由人工处理。不会有错误的 AI 回复让情况更糟。

---

### 场景 D：回访潜在客户

**输入：**
```
用户："嗨，我昨天问过数字营销套餐。有价格更新吗？"
```

**路由代理工作：**
1. 垃圾信息评分？0
2. 升级？没有
3. 回访潜在客户？是的，在 CRM 中找到 lead_id
4. 之前状态：发现（DISCOVERY）（昨天未完成）
5. 决策：**发现（DISCOVERY）**（重新进入）→ 用旧数据委托给发现代理

**输出给用户：**
```
代理："您好 [姓名] 先生，很高兴再次聊天。昨天我们讨论了您在雅加达的业务。让我继续 — 请问，您每月的目标客户数量是多少？"
```

**商业价值：** 不需要再次询问姓名、地点、业务。从昨天继续。销售不会被看作"忘记"客户。

---

### 场景 E：跑题（恶作剧聊天）

**输入：**
```
用户："你好美女，你是谁？"
```

**路由代理工作：**
1. 垃圾信息评分？10（轻微跑题）
2. 垃圾信息评分 < 100 → 不屏蔽
3. 升级？没有
4. 决策：**发现（DISCOVERY）** 但引导回来

**输出给用户：**
```
代理："您好！我是 Ucok，来自 NuxHub 的助手。我可以帮助您处理数字营销需求。有什么我可以帮您的吗？"
```

**商业价值：** 恶作剧聊天不会到达销售。被引导回商业话题。如果继续恶作剧，垃圾信息评分会上升。

## 5. 如何使用路由代理（分步指南）

### 步骤 1：安装技能

将技能文件复制到 Hermes 文件夹：
```bash
mkdir -p ~/.hermes/skills/nuxhub-router
cp agents/router/SKILL.md ~/.hermes/skills/nuxhub-router/
```

### 步骤 2：确保支持文件存在

路由代理需要 2 个配置文件：
```
/shared/spam_gate.yaml        ← 垃圾信息评分规则
/shared/escalation_gate.yaml   ← 升级规则
```

确保这些文件存在于您的项目文件夹中。

### 步骤 3：向路由代理发送输入

以此格式发送输入：
```yaml
lead_id: null                   # null = 新客户，"xxx" = 老客户
raw_message: "你好，我需要帮助..."
spam_score: 0                   # 来自垃圾信息门（在路由代理之前运行）
spam_flags: []
escalate_flag: false            # 来自升级门
escalate_reason: null
long_memory: null               # 来自 CRM（如果是回访潜在客户）
```

### 步骤 4：读取路由代理输出

路由代理生成工件：
```yaml
state_assignment: "DISCOVERY"   # 或 BLACKLIST、HUMAN_ESCALATION 等
delegated_agent: "discovery"     # 下一步应该运行哪个代理
reason: "新客户，垃圾信息评分 0，无升级"
is_returning_lead: false
```

### 步骤 5：运行下一个代理

根据 `delegated_agent` 运行下一个代理：
- `discovery` → 加载技能 nuxhub-discovery
- `null` → 停止（用于 BLACKLIST 或 HUMAN_ESCALATION）

## 6. 可配置设置（无需编码）

### 6.1. 更改垃圾信息阈值

编辑 `shared/spam_gate.yaml`：
```yaml
blacklist:
  score: 100    # ← 更改此数字。默认 100。越低越敏感。
```

### 6.2. 添加升级关键词

编辑 `shared/escalation_gate.yaml`：
```yaml
angry:
  keywords: ["angry", "upset", "disappointed", "terrible"]
  # ← 在此处添加新关键词
```

### 6.3. 更改升级回复

编辑 `shared/escalation_gate.yaml`：
```yaml
angry:
  message_to_lead: "我们深表歉意，我将为您转接..."
  # ← 更改此文本以匹配您的品牌声音
```

### 6.4. 更改新客户的初始状态

编辑 `agents/router/SKILL.md` 中的"新客户"部分：
```
IF lead_id == null:
    → state_assignment: DISCOVERY    # ← 如需可更改为其他状态
```

## 7. 需要监控的指标与 KPI

| 指标 | 计算方式 | 目标 |
|---|---|---|
| **垃圾信息屏蔽率** | 黑名单 / 总聊天数 | < 5%（如果太高，垃圾信息门太敏感） |
| **升级率** | 升级到人工 / 总聊天数 | < 15%（如果高，问题太多或 AI 不够智能） |
| **回访潜在客户识别率** | 识别的回访 / 回访聊天数 | > 90%（如果低，CRM 未工作） |
| **路由准确率** | 正确路由 / 总路由 | 通过样本审查手动测量 |
| **路由时间** | 路由代理输入到输出时间 | < 2 秒 |

## 8. 常见问题（商业问题）

### Q：路由代理懂中文吗？混合语言？
**A：** 懂。垃圾信息门和升级门中的关键词可以编辑以添加任何语言。容忍拼写错误和混合语言。

### Q：如果有人只发"嗨"，会被当作垃圾信息吗？
**A：** 不会。例外规则："新客户发的短消息（1-3 个词）= 容忍。" 所以"嗨"会进入发现阶段，不会被屏蔽。

### Q：如果老年人打字很奇怪，会被屏蔽吗？
**A：** 不会。例外："拼写错误或混合语言 = 不懂技术或老年人，不是垃圾信息。"

### Q：这可以用于 WhatsApp / 微信 / 抖音？
**A：** 可以。路由代理不关心平台。它只读取 `raw_message`。平台（微信、抖音、WhatsApp）= 发送消息给路由代理的网关。只需安装正确的网关。

### Q：如果有人在群聊中生气，会被升级吗？
**A：** 会的，只要检测到升级关键词。但对于群聊，升级通常转给群管理员，而不是销售经理。

### Q：如果有人假装想买但实际上是垃圾信息发送者？
**A：** 垃圾信息评分是累积的。每轮都计算。某人可以从"你好，我想买"开始（评分 0），然后第 5 轮开始发垃圾信息。评分累积，最终被屏蔽。所以垃圾信息发送者不能长期停留。

### Q：路由代理可以用于数字营销以外的业务吗？
**A：** 可以。更改 `shared/spam_gate.yaml` 和 `shared/escalation_gate.yaml` 中的配置。但路由逻辑（NEW_LEAD → DISCOVERY）对所有业务保持不变。

## 9. 摘要（一页纸）

**路由代理 = 你线上商店的前门。**

它做 5 件事：
1. **过滤垃圾信息** → 屏蔽垃圾信息发送者，销售不受干扰
2. **检测问题** → 生气的人直接转给人工
3. **识别老客户** → 不再问"您叫什么名字？"
4. **引导到正确的销售** → 热门转给成交，温暖转给培育，冷门转给跟进
5. **保存一切** → 每次聊天都有记录，可审计

**你不需要编码来更改行为。** 编辑 YAML 文件，更改关键词，更改阈值。路由代理会重新读取。
